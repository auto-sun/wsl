API Overview

项目接口简要文档

当前阶段所有接口均基于 mock 数据或占位逻辑实现，已明确区分演示链路与未来真实模型、真实设备、真实时序数据的接入边界。

打开 API 根入口 查看健康检查
统一返回格式

Response Format

{"code": 0, "message": "ok", "data": {...}}

前端页面统一通过 `apiClient` 调用接口，并按该结构解析数据与错误提示。

登录与权限

Auth Placeholder

当前阶段采用基础登录校验占位。HTML 页面需先登录访问，API 为方便演示与调试仍默认开放。

后续可切换到 Django Auth、JWT、单点登录或更细粒度权限控制。

驾驶舱

Dashboard

GET /api/dashboard/overview

返回首页概览卡片、趋势图、热力地图、告警、快捷入口和无人机巡航占位数据。

长势监测

Monitoring

GET /api/monitoring/growth-summary

GET /api/monitoring/heatmap-data

返回长势总览、趋势、NDVI、巡检记录、热力地图数据，并预留多光谱/遥感接入位置。

病虫害检测

Diagnosis

POST /api/diagnosis/upload

GET /api/diagnosis/history

保存上传图片并返回 mock 推理结果，结构已与未来 PyTorch / ONNX / TensorFlow 模型接入兼容。

处方策略

Decision

GET /api/decision/plans

POST /api/decision/generate

POST /api/decision/dispatch

返回策略列表、生成新 mock 策略，并执行“下发预留”流程，不连接真实控制器。

设备管理

Devices

GET /api/devices/status

GET /api/devices/list

GET /api/devices/logs?device_code=CAM-01

POST /api/devices/test

POST /api/devices/command

返回设备状态、日志和占位动作结果，明确说明当前未接真实硬件与真实 MQTT Broker。

健康检查

Infra Placeholders

GET /api/health

返回 MySQL、Redis、InfluxDB、MQTT、AI 服务的当前占位配置，用于演示系统扩展骨架。

Mock 数据说明

Current Mock Boundary

当前图表、地图、设备状态、策略结果、病虫害推理结果均来自后端 mock 数据与占位服务。

上传检测会真实保存图片并创建记录，但诊断结果仍由 `services/ai/inference.py` 生成。

未来扩展

Future Integration

已预留智能决策模型、病虫害识别模型、MQTT topic、Redis 状态缓存、InfluxDB 时序库、WebSocket / 轮询实时推送扩展点。

当前页面展示的是“可演示第一版”，不是实时在线控制系统。
